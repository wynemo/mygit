滚动的时候，有动态的对齐，要把一样的文本块对齐
如何动态的，就是差异的地方，确实没法做到对齐嘛，因为就是修改的
我们来看一个3-ways view的例子（这只是我想到的一个例子，还有可能其他情况）
比如右边比中间多了几行的这个区域，过了这个区域，下面的文本是相同的，就要保持对齐

怎么保持对齐的呢，就是拖动过程中，右边变化，中间不变，等待右边的多的行滚动完
