两个比较差异的文本框，在滚动时，要注意对齐，就是那些相同的部分要对齐
当然不是每时每刻都对齐，而是说差异的部分，往下滚动时，就要对齐一下同步的地方；
举个例子，比如右边比中间多了几行的这个区域，过了这个区域，下面的文本是相同的，就要保持对齐

怎么保持对齐的呢，就是拖动过程中，右边变化，中间不变，等待右边的多的行滚动完

现在我们滚动这块是按百分比来的对吧？
问题是这种和diff没有任何关联，我希望的是不同的的代码块，
在拖动以后，在某一个时间点，能做一些补偿，让相同的代码块对齐；
我举一个例子，右边相对左边在插入一行，我们拖动到这里时，
这样后面的相同的相同的行就要沉下去一样，对吧，但是接着拖，
我希望左边不动了，让右边上升一行，这样就动态对齐了，以此类推

所在的编辑器滚动，为用户的输入，不一定是左是右，作为用户输入
